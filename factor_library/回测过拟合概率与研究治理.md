# 回测过拟合概率与研究治理

## 类型

研究质量控制模块，不是单一 alpha 因子。

## 目标

识别策略、因子、参数和模型搜索过程中的回测过拟合风险，并在进入正式回测或上线前建立最低研究审计门槛。

## 输入

- 策略候选绩效矩阵：时间 `T` 行，策略/参数/模型候选 `N` 列。
- 绩效指标：Sharpe、Sortino、Calmar、年化收益、均值收益等。
- 研究清单：假设、资产池、基准、数据来源、信息可得时点、信号定义、仓位规则、执行价格、成本、样本外计划、参数敏感性和失败模式。

## CSCV/PBO定义

1. 将时间序列切成偶数个连续区块。
2. 每次选一半区块作为训练集，剩余区块作为测试集。
3. 在训练集中按目标指标选择最优候选。
4. 在测试集中观察该训练赢家的相对排名。
5. 若训练赢家在测试集中落入后半区，则记为一次过拟合。
6. `PBO = 过拟合次数 / 有效组合次数`。

本库采用明确排名约定：`test_rank_best1 = 1` 表示样本外最好，`relative_rank_best1 = test_rank_best1 / (N + 1)`。当 `relative_rank_best1 > 0.5` 时，代表训练赢家样本外落入较差半区。

## 解释

- `PBO < 0.2`：候选集合暂时可接受，但仍需独立样本外和成本检验。
- `0.2 <= PBO < 0.5`：存在明显选择偏差，应缩小参数空间、加入经济约束或增加样本外。
- `PBO >= 0.5`：历史最优很可能来自过拟合，不应直接上线。
- 训练指标明显高于测试指标：样本内优化过强。
- 选择频率过于分散：候选之间缺少稳定优势。
- 选择频率过于集中但测试排名差：单一候选可能被少数样本期推动。

## 使用位置

- 参数网格搜索之后。
- 多模型比较之后。
- 大量因子挖掘之后。
- 机器学习调参之后。
- 技术指标择时和 ETF 轮动参数选择之后。

## 失效与注意

- CSCV 会重组时间区块，不能完整保留路径依赖；对强路径依赖策略仍需完整滚动回测。
- 如果候选策略本身没有经济逻辑，低 PBO 也不代表可投资。
- 样本期过短或策略数量过少时，PBO 不稳定。
- 评价指标必须匹配策略目标；指数增强应更多关注超额收益和信息比率，绝对收益择时才看 Sharpe/Calmar。
- PBO 只是过拟合诊断，不替代交易成本、容量、风险暴露、拥挤度和实盘模拟。
