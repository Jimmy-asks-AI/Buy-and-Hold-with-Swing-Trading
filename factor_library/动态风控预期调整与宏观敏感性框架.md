# 动态风控、预期调整与宏观敏感性框架

## 模型卡 1：动态因子敞口边界

- 目标：通过风控约束实现因子择时，而不是只改变 alpha 权重。
- 输入：因子收益预测、因子平均正收益、因子平均负收益、风险厌恶度。
- 规则：
  - 预测为正：下限 0，上限按平均负收益和风险厌恶度计算。
  - 预测为负：上限 0，下限按平均正收益和风险厌恶度计算。
- 适用：指数增强、风险控制组合、因子暴露受限组合。
- 风险：因子收益预测错误会导致错误放宽敞口；平均正/负收益窗口需要稳健性测试。
- 代码：`risk_model_tools.dynamic_factor_exposure_bounds`。

## 因子卡 1：时间序列标准化预期调整

- 类型：分析师一致预期、基本面变化。
- 原始指标：预期 ROE、预期净利润同比增速、预期两年复合增长率。
- 定义：`(当前预期 - 过去 N 个月均值) / 过去 N 个月波动率`。
- 默认窗口：12 个月。
- 方向：越高越好。
- 数据治理：预测类型 1/2 保留，类型 3/4 可填 0。
- 风险：覆盖度、预测可得日、行业差异、预期修订滞后。
- 代码：`analyst_expectation_factors.expectation_adjustment_factor`。

## 模型卡 2：宏观敏感性 MacroBeta

- 目标：刻画股票收益对宏观变量变化的敏感方向和程度。
- 输入：个股收益、宏观变化指标、市场收益和风格控制。
- 计算：滚动回归 `ret ~ macro_change + controls`，宏观项系数为 MacroBeta。
- 使用：不能直接买高 beta；应形成 `macro_score = MacroBeta * expected_macro_change`。
- 稳定性门槛：滚动 MacroBeta 需要有跨期稳定性，低稳定性宏观指标不入库。
- 风险：宏观数据发布滞后、修正、非平稳、风格污染。
- 代码：`macro_sensitivity_models.rolling_macro_sensitivity`、`macro_score_factor`。

## 因子卡 2：宏观不确定性 beta

- 类型：宏观不确定性风险暴露。
- 输入：中国 EPU 指数变化、个股收益、市场/SMB/HML 等控制变量。
- 定义：滚动回归中 EPU 变量的暴露系数。
- 方向：报告中 A 股为正向，沪深 300 中最强。
- 使用：优先在大市值股票池和指数增强中验证。
- 必做控制：市值、PB、ROE、换手、反转、特质波动、行业。
- 代码：`macro_sensitivity_models.uncertainty_beta_factor`。

## 入库标准

1. 动态敞口模型必须同时记录收益预测来源和风险边界来源。
2. 预期调整因子必须按可得日构造，且拆分可靠预测与填充预测。
3. MacroBeta 必须使用平稳宏观变量，并剥离常见风格。
4. 宏观得分必须有真实可得的宏观方向预测，不能用未来宏观变化。
5. EPU beta 必须分股票池验证，不能把沪深 300 结论外推到全市场。
