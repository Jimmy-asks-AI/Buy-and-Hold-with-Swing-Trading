# 空头收益转化与敞口预算

## Purpose

把“空头端有效”的因子转化为长-only 组合可用的信息，并用因子敞口上限控制收益和风险的交换。

## Single-Factor Conversion

```text
short_contribution = abs(short_group_excess) / long_short_return
```

若 `short_contribution` 高且 `long_group_excess` 弱：

- 不直接做多多头组。
- 用逆向剔除删除短端股票。
- 在剩余股票池内再运行主模型。

## Multi-Factor Conversion

```text
composite_ir = (w' * mean_ic) / sqrt(w' * cov_ic * w)
optimal_direction = inv(cov_ic) * mean_ic
```

使用条件：

- 因子值先去极值、标准化。
- 因子间需正交或至少做相关性控制。
- 权重估计需要滚动窗口和 shrinkage，避免 IC 协方差不稳定。

## Exposure Budget

组合优化中的因子敞口上限是风险预算：

```text
active_exposure = sum((portfolio_weight - benchmark_weight) * factor_exposure)
abs(active_exposure) <= exposure_cap
```

原则：

- 上限越高，潜在收益越高，但回撤、波动和跟踪误差也更高。
- 应监控实际敞口，而非只记录预设上限。
- 不同指数、行业结构和因子数量对应不同合适上限。

## Rolling Cap

```text
next_cap = rolling_mean(abs(realized_exposure), window).shift(1)
```

用途：

- 避免用历史最优参数过拟合。
- 让敞口上限随指数结构、因子体系和优化约束自适应。

## Failure Modes

- 逆向剔除过多股票，导致股票池过窄或行业偏离。
- IC 协方差估计不稳定，复合 IR 权重过度集中。
- 因子正交顺序影响残差定义。
- 提高敞口上限后收益上升但风险恶化更多。
- 只看预设敞口上限，不看实际敞口。
