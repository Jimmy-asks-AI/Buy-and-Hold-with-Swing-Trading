# Factor: 资金流向与大单行为因子

Domain: Order Flow / High-Frequency Alpha
Universe: A 股，可扩展到流动性足够的股票池
Frequency: Daily or intraday aggregation
Data Fields: order amount, trade amount, trade count, minute return, order-book bid/ask volume, float shares, buy/sell side, active/passive flag, timestamp, abnormal trading flag

## Rationale

资金流向和大单成交可以刻画市场短期微观供求。大额主动买入、主力净流入、开盘主力净流入可能反映机构或大资金建仓行为；异常交易和过度追涨则可能带来后续反转风险。

## Formula

```text
main = large_order + super_large_order
main_net_inflow = main_buy_amount - main_sell_amount
main_inflow_rate = main_net_inflow / (main_buy_amount + main_sell_amount)
open_main_inflow_rate = open_main_net_inflow / open_main_total_amount
close_main_inflow_rate = close_main_net_inflow / close_main_total_amount
big_buy_ratio = big_buy_amount / total_trade_amount
big_sell_ex_big_buy_ratio = sell_big_order_without_big_buy_amount / total_trade_amount
dynamic_big_order_threshold = mean(order_amount) + N * std(order_amount)
big_buy_amount_ratio = dynamic_big_buy_amount / total_trade_amount
big_buy_minus_sell_ratio = big_buy_amount_ratio - big_sell_amount_ratio
buy_order_concentration = sum(buy_order_amount_k^2) / total_trade_amount^2
sell_order_concentration = sum(sell_order_amount_k^2) / total_trade_amount^2
active_buy_ratio = active_buy_amount / total_or_session_amount
active_net_buy_intensity = mean(active_buy_amount - active_sell_amount) / std(active_buy_amount - active_sell_amount)
avg_amount_per_trade = sum(trade_amount) / sum(trade_count)
outflow_avg_amount_ratio = avg_amount_per_trade_on_down_minutes / avg_amount_per_trade
big_minute_net_inflow_ratio = big_minute_net_inflow / total_trade_amount
big_minute_driven_return = prod(1 + minute_return_on_big_trade_minutes) - 1
net_bid_change_rate = (sum(delta_bid_volume) - sum(delta_ask_volume)) / float_shares
net_bid_change_mean = mean(net_bid_change_rate by session)
net_bid_change_vol = std(net_bid_change_rate by session)
net_bid_change_skew = skew(net_bid_change_rate by session)
order_trade_corr = corr(minute_return, net_bid_change_rate)
unexpected_return = minute_return - expected_minute_return
informed_buy_amount = active_buy_amount where unexpected_return < 0
informed_sell_amount = active_sell_amount where unexpected_return > 0
informed_net_buy_ratio = (informed_buy_amount - informed_sell_amount) / total_or_session_amount
big_buy_without_big_sell_ratio = amount(buy_order_big and not sell_order_big) / total_amount
big_sell_without_big_buy_ratio = amount(sell_order_big and not buy_order_big) / total_amount
big_buy_and_big_sell_ratio = amount(buy_order_big and sell_order_big) / total_amount
```

订单大小参考：

- small: amount < 40,000
- medium: 40,000 <= amount < 200,000
- large: 200,000 <= amount < 1,000,000
- super_large: amount >= 1,000,000

## Preprocessing

- 剔除 ST、停牌、涨跌停不可交易样本。
- 对日截面 winsorize 和 z-score。
- 行业和市值中性化。
- 检查与换手、动量、波动的残差表现。
- 交易异动股票剔除或使用哑变量正交。
- 从逐笔成交还原买卖单时，使用叫买序号和叫卖序号聚合成交金额。
- 使用 BS 标志计算主动买入时，应剔除涨跌停分钟。
- 动态大单阈值需要做参数敏感性测试。
- 分钟成交金额因子需要剔除集合竞价，并处理日内长时间价格不动的股票日。
- 盘口委托快照因子必须处理最优价变动导致的档位迁移，不能只机械比较同一档位数量。
- 净委买变化率应除以流通股本，便于不同规模股票横向比较。
- 常用日内时段：开盘后 30 分钟、盘中、收盘前 30 分钟；不同报告口径略有差异，回测必须固定。
- 知情交易因子先用历史分钟收益模型剔除星期、时段和滞后收益效应，再用残差方向筛选主买主卖。
- 知情主买、知情主卖必须保留原始方向和报告调整方向，避免后续误读。
- 大买、大卖成交金额占比应按对手方是否为大单进一步拆分。
- 逐笔过滤重构 K 线时，要记录保留成交额占比，避免过滤过度导致信息不足。

## Evaluation

- IC: 1、3、5、10、20 日未来收益。
- Rank IC: 同上。
- ICIR: 分年度和滚动窗口。
- Group monotonicity: 5 或 10 组分层。
- Long-short return: 扣除高换手成本。
- Turnover: 必须报告。
- Capacity: 用成交额、冲击成本和涨跌停可交易性约束。

## Robustness

- 不同行业、不同市值区间分别测试。
- 不同持仓周期测试。
- 不同调仓频率与不同计算窗口分开测试。
- 不同数据商字段口径交叉验证。
- 剔除公告、异动和龙虎榜股票后复测。
- 对量价控制变量残差复测。
- 对前 1 档、前 5 档、前 10 档和全部档位分别测试。
- 对月度、半月、周度调仓分别测试，并把计算窗口和调仓周期显式记录。
- 对逐笔大单因子测试 1-3、4-6、7-9、10-21 等区间收益，检查预测力衰减。
- 单因子有效后，还要放入已有组合模型测试边际贡献。

## Failure Modes

- 信号半衰期短，持仓过长会失效。
- 高频数据质量问题导致伪信号。
- 成本、冲击、涨跌停限制会严重侵蚀收益。
- 大资金买入并不总是利好，交易异动股票长期可能反转。
- 资金流向因子可能只是换手、流动性或市值暴露。
- BS 标志在涨停/跌停分钟会产生方向解释偏差。
- 高频毛收益可能完全来自未扣成本的高换手。
- 大单净流入和大单驱动上涨可能是对倒或冲击后的反转，不应机械解释为利好。
- 盘口挂单可撤单，净委买变化率可能被虚假挂单污染。
- 委托成交相关性主要是个股截面因子，直接迁移到指数择时会丢失横截面信息。
- 收益若主要来自空头端，在 A 股长线多头组合中只能作为排除、降权或做 T 辅助信号。
- 知情交易残差模型过简会把普通日内模式误识别为信息交易。
- 收盘前知情主买可能是过度反应信号，不能按名称直接解释为正向买入。
- 大单因子预测力衰减快，持有期过长会稀释信号。
- 重构 K 线因子与大单因子可能高度重叠，组合时存在重复计量。

## Implementation Notes

- 先把资金流因子作为短周期 alpha 模块，不纳入长线估值因子同一评价周期。
- 使用时应有换手预算和持仓上限。
- 可作为做 T 或短期增强信号，但需要严格成本模型。
- 若用于中低频做 T，优先测试开盘后/盘中净主买强度，而不是直接追随尾盘主买占比。
- 平均单笔流出金额占比可作为短期冲击补偿/反转候选因子。
- 前 1 档开盘后净委买变化率更接近真实交易意愿，深档位信号通常更弱。
- 收盘前委托成交相关性在正交后仍较稳，可用于识别“下跌中买盘托底”。
- 长线高分红低估值框架中，本模块应定位为微观择优和交易节奏模块，不替代估值、盈利质量和宏观利率判断。
- 开盘后知情净主买可作为短周期正向信息交易候选；收盘前知情主买更适合作为过热或反转过滤。
- 大单买入和大单卖出不对称：大卖对小买更偏负面，大买被大卖承接仍可能偏正面。
- 逐笔因子进入组合前必须做相关性筛选，例如主买强度和开盘知情主卖可能高度相关。
