# 深度学习高频因子改进检查清单

创建日期：2026-05-23  
来源资料：海通证券《选股因子系列研究（七十七）——改进深度学习高频因子的9个尝试》  
对应代码：`strategy_lab/lob_deep_highfreq_models.py`

## 必查 9 项

| 模块 | 研究动作 | 通过标准 |
|---|---|---|
| 正交训练 | 对行业、风格、低频技术因子加正交层或残差约束 | 正交后仍有 IC 和组合增量 |
| 特征压缩 | 按经济逻辑压缩高频特征 | 样本外表现不显著下降 |
| 特征标准化 | 比较 raw、时序标准化、截面标准化 | 横截面选股优先截面标准化 |
| 输入频率 | 比较 30 分钟、10 分钟等频率 | 高频率必须带来净增益 |
| 切分比例 | 调整训练集/验证集长度 | 不能牺牲验证稳定性换取近期过拟合 |
| 环境变量 | 加入市场状态、波动、流动性等变量 | 必须证明分状态改善，而非简单拼接 |
| 预测目标 | 比较原始收益、超额收益、风险调整超额收益 | 与最终组合收益目标一致 |
| 训练窗口 | 比较短窗口和长窗口 | 长窗口应提升稳定性或样本外收益 |
| 模型复杂度 | 比较参数量和层数 | 复杂度提升必须有样本外收益增量 |

## 推荐研究流程

1. 构造高频序列张量，只使用预测时点之前的数据。
2. 对输入特征做横截面标准化，并保留 raw/时序标准化作为对照组。
3. 使用滚动训练/验证切分，不做随机打乱。
4. 训练原始模型和正交约束模型。
5. 分别输出原始预测、正交预测和风险调整目标预测。
6. 对预测结果做行业、市值、估值、换手、反转、波动等控制。
7. 在中证 500/中证 1000/全 A 股票池分别做增量组合测试。
8. 扣除交易成本，并设置换手约束。
9. 报告分年度、分市场状态、多随机种子和容量结果。

## 失败判定

以下任一情况出现时，不应把模型输出直接入库：

- 正交后 IC 消失。
- 只在训练集或单一年份有效。
- 提升来自更高换手但成本后收益下降。
- 模型复杂度增加但样本外没有改善。
- 高频率输入导致因子自相关下降、换手上升且收益不改善。
- 环境变量只在事后分组中有效。
