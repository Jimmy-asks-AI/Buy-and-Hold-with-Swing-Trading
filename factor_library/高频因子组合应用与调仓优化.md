# 高频因子组合应用与调仓优化

## Purpose

将短周期高频信号嵌入低频组合，不直接假设高频信号可通过高换手获得净收益。核心用途是收益预测增强、空头风险过滤、后置剔除和延迟调仓。

## Inputs

- 低频组合当前权重。
- 低频组合目标权重。
- 短周期高频因子或复合高频信号。
- 成交成本、冲击成本、涨跌停和停牌约束。
- 未来 1/3/5/10/20 日收益标签，用于信号衰减测试。

## Composite Signal

```text
hf_composite = weighted_sum(zscore(sign_i * factor_i))
```

权重来源：

- 等权：用于初始研究，避免过拟合。
- 历史 IC 或 Rank IC 加权：用于有足够滚动样本时。
- 风险约束加权：降低高度相关或换手过高因子的权重。

高频因子必须先统一方向：数值越高表示短期越好。负向因子需要乘以 -1。

## Delay Rebalance Rule

在低频调仓日 T：

```text
planned_trade = target_weight - current_weight
delay_sells = planned sells with highest hf_composite
delay_buys = planned buys with lowest hf_composite
delayed_notional = min(delay_budget, sell_capacity, buy_capacity)
immediate_trade = planned_trade - delayed_trade
```

解释：

- 高信号的计划卖出股票，短期仍可能强，因此延迟卖出。
- 低信号的计划买入股票，短期风险高，因此延迟买入。
- 延迟买卖金额要平衡，避免引入现金或杠杆暴露。

参数：

- `delay_ratio`：延迟仓位比例，初始可测 1%、3%、5%、10%。
- `delay_days`：延迟天数，初始可测 1、3、5、10 日。
- `budget_base`：可用总仓位或计划单边换手作为基准。

## Short-Leg Filter

若高频信号主要由空头端贡献：

```text
short_dummy = hf_composite <= cross_section_quantile(hf_composite, 5%)
```

使用方式：

- 前置剔除：从股票池中删除短端股票，再重新优化。
- 后置剔除：先优化组合，再把短端股票权重置零并按原权重比例重分配。
- 降权：不完全剔除，只把短端股票权重乘以折扣系数。

后置剔除更容易衡量边际贡献，但要防止行业和个股权重约束被破坏。

## Evaluation

- 信号层：1/3/5/10/20 日 IC、Rank IC、分组收益、单调性。
- 组合层：年化收益、年化超额、最大回撤、信息比率、胜率、换手。
- 交易层：手续费、印花税、滑点、冲击、成交容量、涨跌停不可交易。
- 稳定性：年份、市场风格、行业、市值、流动性分层。
- 边际贡献：加入高频模块前后对原模型的增量表现。

## Failure Modes

- 高频因子半衰期短，延迟天数过长会失效。
- 调仓价格使用均价或 VWAP 会高估真实可交易性。
- 基础组合换手太低时，可延迟交易池不足。
- 延迟比例太高会稀释信号差异并扰动原模型。
- 空头端因子被误用为多头买入信号。
- 组合剔除后若不重新检查行业、市值和个股约束，会制造新的风险暴露。
- 机器学习表达式可能挖到数据供应商或样本期特有噪声。
