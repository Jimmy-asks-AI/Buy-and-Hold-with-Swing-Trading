# 关系网络、资产稳定、价量波幅与被动扩张框架

## 因子卡 1：股价相关性网络

- 类型：图网络、另类数据、价量结构。
- 输入：过去 N 日个股收益矩阵。
- 构边：保留绝对相关性位于高分位的股票对。
- 输出：度、中心性、邻居收益溢出。
- 方向：
  - 度和中心性：报告中偏正向。
  - 短周期邻居收益：偏反转，使用 `-neighbor_return`。
  - 长周期邻居收益：可测试动量。
- 必做控制：行业、市值、反转、换手率、波动率、流动性。
- 代码：`correlation_network_factor_snapshot`。

## 因子卡 2：主营业务相似网络

- 类型：基本面关系网络。
- 输入：主营产品/业务分类，可带收入权重。
- 输出：业务网络度、中心性。
- 风险：高度接近行业分类，行业中性后可能大幅衰减。
- 代码：`business_similarity_network_features`。

## 因子卡 3：资产增长稳定性

- 类型：基本面稳定性、资产负债表质量。
- 定义：资产增长率的滚动波动率。
- 推荐口径：流动资产增长波动率，近期 1 年窗口优先。
- 方向：波动率越低越好，即 `-asset_growth_volatility`。
- 扩展：行业内中位数可做行业轮动信号。
- 代码：`asset_growth_volatility_factor`、`industry_asset_stability_factor`。

## 因子卡 4：资本结构变化

- 类型：杠杆变化、资本结构。
- 定义：股东权益比率、资产负债率、长期负债率、股东权益/固定资产的同比或环比变化。
- 方向：
  - 资产负债率变化：报告中偏正向。
  - 股东权益比率变化：方向相反。
- 使用限制：按行业、盈利状态验证；成长行业更强。
- 代码：`capital_structure_change_factors`。

## 因子卡 5：价格振幅与换手波幅

- 价格振幅：`rolling_high / rolling_low - 1`，正向，但必须正交传统价量因子。
- 换手波幅：`rolling_max(turnover) - rolling_min(turnover)`，负向。
- 窗口：价格振幅 1-6 个月可测，2-3 个月优先；换手波幅优先 1 个月。
- 持有期：技术因子属性强，超过 3 个月需重新验证。
- 代码：`price_amplitude_factor`、`turnover_amplitude_factor`、`orthogonalize_by_date`。

## 状态卡：被动产品扩张

- 类型：市场结构状态、alpha 风险状态。
- 定义：被动产品份额或规模占比的快速上升。
- 用途：扩张期降低反转、换手率等反转型因子权重；监控指数增强超额收益压力。
- 风险：只说明相关性，不能直接证明被动扩张导致 alpha 衰减。
- 代码：`passive_expansion_regime`、`alpha_performance_by_passive_regime`、`passive_regime_factor_weight_adjustment`。

## 入库标准

1. 网络因子必须报告与传统因子的相关性及正交后 IC。
2. 资产增长稳定性必须区分总资产、流动资产、非流动资产。
3. 资本结构变化必须按行业和盈利状态分层。
4. 价量波幅必须报告观察期、持有期衰减和正交后收益。
5. 被动扩张状态只用于风控和权重调整，不能单独作为交易信号。
