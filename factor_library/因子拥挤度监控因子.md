# Factor: 因子拥挤度监控因子

Domain: Factor Timing / Risk Control
Universe: A 股横截面因子、多因子组合、指数增强组合
Frequency: Monthly preferred; weekly possible after validation
Data Fields: factor score, stock returns, valuation, market return, residual return, industry/style exposure

## Rationale

当大量资金追逐同一因子时，因子多头端可能变贵、成分同涨同跌增强、前期收益过高、波动上升，随后因子收益可能走弱或出现回撤。拥挤度用于监控这种内生风险。

## Formula

单项指标：

```text
valuation_spread = log(valuation_long / valuation_short)
pairwise_corr = avg_corr(long stocks, long portfolio) +/- avg_corr(short stocks, short portfolio)
long_term_return = cumulative_return(long_short_factor, lookback_window)
volatility_ratio = vol(long or long_short factor) / vol(short or market)
asset_absorption_ratio = sum(top_n_pca_eigenvalues) / sum(all_pca_eigenvalues)
institution_holding_amount_ratio = fund_holding_amount_in_long / total_fund_holding_amount
institution_holding_market_cap_ratio = fund_holding_amount_in_long / long_leg_market_cap
```

复合指标：

```text
crowding = mean(ts_zscore(indicator_k))
```

默认解释为数值越高越拥挤；若某一指标在特定因子上方向相反，必须单独设置方向。

## Preprocessing

- 对极值组使用前后 10% 股票。
- 每个指标按因子自身历史做时间序列标准化。
- 估值指标需处理负值、极端值和缺失值。
- 配对相关性可使用原始收益或特质收益，但两者不能混用解释。
- 使用正交因子时，要同时保留原始因子版本做对比。
- 资产集中度需要固定 PCA 窗口和主成分数量。
- 机构持仓指标需要按实际披露日滞后处理，不能用报告发布日期前不可得数据。

## Evaluation

- IC: 不适用，拥挤度是时间序列因子。
- Rank IC: 不适用。
- ICIR: 可用拥挤度与未来因子收益的滚动相关替代。
- Group monotonicity: 将拥挤度分桶，观察未来收益、波动和最大回撤。
- Long-short return: 高拥挤减配、低拥挤增配后的因子择时收益。
- Turnover: 拥挤度调权带来的因子权重换手。
- Capacity: 高拥挤时通常同步检查成交额和持仓集中度。

## Robustness

- 分因子校准阈值。
- 分牛熊市和大小盘风格阶段测试。
- 对 `+1/-1` 阈值做敏感性分析。
- 对 6、12、18、24 个月未来窗口分别测试。
- 与外生宏观择时变量分开归因。
- 在全市场、宽基指数池和实盘可投池分别校准阈值。

## Failure Modes

- 高拥挤后可能继续上涨，过早减配会损失趋势收益。
- 低拥挤可能是基本逻辑失效，而不是便宜。
- 特质收益模型错误会污染配对相关性。
- 极端拥挤样本少，回撤概率统计容易不稳。
- 过度使用拥挤度会变成事后追涨杀跌的反向数据挖掘。
- 指标之间高度相关时，复合打分会重复计量同一拥挤来源。

## Implementation Notes

- 复合拥挤度只作为风险预算和因子权重调整输入，不直接替代因子本身。
- 对 long-only 或红利类策略，应优先监控多头端估值、相关性、换手和成交拥挤。
- 实盘规则应采用缓慢调权，例如拥挤度高于阈值后逐步降低因子倾斜强度。
- 持仓类拥挤度适合季度级复核，不适合作为月内快速调仓信号。
