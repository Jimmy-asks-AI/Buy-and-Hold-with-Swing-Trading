# 技术指标信号验证框架

## 类型

技术指标信号验证工具，不是完整交易策略。

## 研究目标

判断某个技术指标信号出现后，未来若干交易日收益分布是否与无信号或反向信号不同。

## 最小流程

1. 读取复权行情。
2. 只用当前及过去数据计算指标。
3. 用交叉、阈值或排序生成信号。
4. 计算未来收益标签。
5. 按信号分组统计未来收益。
6. 输出均值、中位数、胜率、分位数和样本量。

## 必须区分

信号验证回答：

```text
信号出现后未来收益是否有统计差异？
```

完整回测回答：

```text
按规则交易、扣成本、处理不可交易后，资金曲线是否优于基准？
```

## KDJ示例

核心公式：

```text
LOW_N = rolling min(low, N)
HIGH_N = rolling max(high, N)
RSV = (close - LOW_N) / (HIGH_N - LOW_N) * 100
K = smooth(RSV)
D = smooth(K)
```

信号：

```text
K 上穿 D -> signal = 1
K 下穿 D -> signal = 0
```

## 过拟合风险

- 指标数量多。
- 参数窗口多。
- 阈值选择多。
- 买卖规则可变。
- 样本区间可选。

因此技术指标研究应默认加入：

- 样本外。
- 分年度。
- 参数敏感性。
- PBO/CSCV。
- 成本后表现。
- 与动量、反转、波动等基础因子的相关性。

## 失败模式

- 技术指标只是价格动量、反转或波动的重包装。
- 信号样本量太小，胜率不稳定。
- 最优参数来自样本内噪声。
- 忽略交易成本后看似有效。
- 个股层面无法成交，或被停牌、涨跌停影响。
