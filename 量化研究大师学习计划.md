# 通用量化研究大师学习计划

定位：先建立通用量化研究能力，不绑定任何单一投资主线。以后你的策略方向可以变，但研究系统、证据标准、代码模板和知识沉淀方式不变。

## 目标

最终要形成一个可调用的“量化研究大师”能力包：

- 能读金融学论文、卖方金工研报、课程材料和代码。
- 能把资料转成研究问题、因子、信号、组合、风控和回测实验。
- 能保存为 `skill`、研究笔记、因子库、策略模板、复现实验代码。
- 下次你可以直接调用该 skill，让它继续读资料、做研究、复现模型或设计策略。

## 总路线

```text
资料索引
-> 主题分类
-> 基础课程精读
-> 研究标准建立
-> 回测闭环
-> 因子库
-> 策略库
-> 组合与风控
-> 论文/研报复现
-> 生产级研究流程
-> skill 化沉淀
```

## 阶段 0：建立资料索引

时间：1-2 天。

动作：

1. 扫描全部文件。
2. 生成 `research_manifest.csv`。
3. 生成 `reading_queue.csv`。
4. 为每个文件打标签和优先级。

标签体系：

- `foundation`：基础概念、课程、研究框架。
- `data`：数据源、清洗、复权、样本构建。
- `backtest`：回测、交易成本、样本外验证。
- `factor`：因子研究、多因子、因子评价。
- `timing`：择时、趋势、反转、技术指标、市场状态。
- `portfolio`：组合构建、优化、风险预算。
- `risk`：风控、回撤、波动、压力测试。
- `execution`：交易执行、换手、滑点、容量。
- `behavioral`：行为金融。
- `sentiment`：投资者情绪。
- `ml`：机器学习、深度学习、AI 金融。
- `fund_etf`：基金、ETF、产品研究。
- `derivatives`：期货、期权、衍生品。
- `macro`：宏观、利率、信用、流动性。

输出：

- `research_manifest.csv`
- `reading_queue.csv`
- 第一批 P0/P1 阅读清单

## 阶段 3：因子研究能力

时间：6-8 周。

主读：

- `资料/卖方金工研报/多因子/`
- `资料/卖方金工研报/海通报告/`
- `资料/因子挖掘.md`
- `资料/投资组合优化与风控.md`

因子大类：

- 估值：PE、PB、PCF、PS、EV/EBITDA、股息率。
- 质量：ROE、ROA、毛利率、现金流质量、盈利稳定性、杠杆。
- 成长：收入增长、利润增长、预期上修。
- 动量：短中长期动量、行业动量、残差动量。
- 反转：短期反转、长期均值回归、超跌反弹。
- 波动：低波、下行波动、Beta、特质波动。
- 流动性：成交额、换手率、Amihud、冲击成本。
- 情绪：新高新低、涨跌停、融资、资金流。
- 宏观：利率、信用、通胀、汇率、期限利差。
- 事件：财报、分红、回购、指数调仓、政策事件。

输出：

- `factor_library.csv`
- `strategy_lab/factor_evaluation.py`
- `strategy_lab/factor_merge_model.py`
- `replication_reports/多因子阶段报告.md`

## 阶段 4：择时与市场状态

时间：6-8 周。

目标：建立通用择时研究框架，不限定任何资产。

方向：

- 趋势：均线、突破、动量、趋势强度。
- 反转：超跌、偏离、均值回归。
- 估值：估值分位、风险溢价、跨资产比价。
- 波动：波动率、下行波动、波动率聚集。
- 流动性：成交额、换手、资金面。
- 情绪：恐慌、拥挤、散户/机构行为。
- 宏观：利率、信用、PMI、通胀、汇率。
- 市场广度：上涨家数、均线以上比例、新高新低。

输出：

- `timing_factor_library.csv`
- `strategy_lab/timing_model_template.py`
- `replication_reports/择时模型阶段报告.md`

## 阶段 5：组合、风控、执行

时间：6 周。

主读：

- 组合优化、风险控制、基金/ETF、交易执行相关材料。

能力：

- 等权、风险平价、最小方差、均值方差、Black-Litterman。
- 行业/风格/个股约束。
- 换手控制、交易成本、滑点、容量。
- 回撤控制、波动目标、止损/止盈、压力测试。
- 归因：资产、行业、风格、因子、择时、选股。

输出：

- `portfolio_lab/portfolio_optimizer.py`
- `portfolio_lab/risk_report_template.md`
- `execution_cost_model.py`

## 阶段 6：论文和研报复现

时间：长期。

顺序：

1. 先复现卖方金工研报。
2. 再复现行为金融和投资者情绪论文。
3. 最后复现 AI 金融论文。

每篇资料都问：

- 研究问题是什么？
- 数据能不能拿到？
- 方法有没有未来函数？
- 结论是否稳健？
- 能否转成因子、信号、组合或风控规则？
- 样本外是否还能成立？

输出：

- `notes/report_notes/`
- `notes/paper_notes/`
- `replication_reports/`
- `strategy_lab/paper_replications/`

## 阶段 7：Skill 化沉淀

需要形成一个可调用 skill：`quant-research-master`。

skill 内应包含：

- `SKILL.md`：触发条件和工作流。
- `references/learning_protocol.md`：学习和消化资料流程。
- `references/research_standards.md`：研究证据标准。
- `references/domain_map.md`：量化研究知识地图。
- `references/output_templates.md`：笔记、因子、策略、复现模板。
- `scripts/build_research_manifest.py`：扫描资料库并生成索引。
- `scripts/init_quant_research_workspace.py`：初始化研究目录和模板。
- `scripts/quick_validate_research_artifacts.py`：检查研究产出是否合格。

## 日常执行规则

每天：

1. 扫一份资料，定标签和优先级。
2. 精读一份 P0/P1 资料。
3. 写一份结构化笔记。
4. 抽取一个因子、信号、模型或风控规则。
5. 能复现则进入代码实验，不能复现则记录原因。

每周：

- 至少 1 篇精读笔记。
- 至少 1 个因子或信号实验。
- 至少 1 个回测/复现报告。
- 至少 1 次失败复盘。

每月：

- 更新因子库。
- 更新策略库。
- 更新研究标准。
- 整理可复用模板。
- 把成熟流程同步进 skill。

## 大师级验收标准

不是“读完多少篇”，而是：

- 有完整资料索引。
- 有稳定阅读队列。
- 有统一研究模板。
- 有可运行回测代码。
- 有 50+ 因子定义。
- 有 10+ 复现报告。
- 有样本内/样本外/滚动验证。
- 有交易成本和容量约束。
- 有组合和风控模块。
- 有失败案例库。
- 有可调用 skill，能在新任务中延续研究能力。
