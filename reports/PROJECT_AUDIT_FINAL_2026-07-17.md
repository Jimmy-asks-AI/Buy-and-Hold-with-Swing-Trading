## 1. 总结论

仓库已经形成了较完整的量化研究知识库和一条失败关闭的当前截面研究链，但尚未形成经过无偏历史验证的可投策略。

- **当前默认系统**：Long Hold Dividend V4，面向 50 万元现金账户、稳定分红股票/红利 ETF、深度低估分批建仓及中低频 T。
- **当前运行结果**：`CASH_NO_ENTRY_SIGNAL`，500,000 CNY 现金、零持仓、零订单。
- **可以信任**：当前数据门禁、价值陷阱否决、仓位约束、订单意图、人工成交记账、基础分红和送转处理，以及输入哈希追踪。
- **历史数据进度**：Gate E2的15类必需数据已有5类通过，分别是全状态股票主表、PIT财务、股票现金分红事件、全状态ETF主表和中美利率；其余10类继续失败关闭。
- **不能信任**：V4 预期收益、Sharpe、回撤、择时有效性和做 T 成本后收益；这些尚无正式 PIT walk-forward 证据。
- **旧系统结论**：旧 HIRSSM 和旧因子工厂只能作为研究历史。已发现会污染绩效的确定性实现缺陷，旧收益不得用于策略选择。
- **多 Agent 结论**：存在职责、输入输出和禁止事项契约，但没有独立上下文和独立运行时，准确描述是“受 Agent 契约约束的单进程规则系统”。

综合评级：**C 级研究系统，D+ 级可投系统**。当前应继续保持研究模式，不应自动交易，也不应报告预期收益。

## 2. 重大问题清单

### P0-1：正式 PIT 历史链不完整，且没有样本外绩效

V4 回测器本身使用逐日持仓、收盘信号和下一开盘成交，不复利重叠标签；同时强制 `promotion_allowed=false`（`strategy_lab/long_hold_v4/backtest.py:1-5, 311-324`）。目前已建立可执行Gate E2，并取得五类合格输入：

- 事件化股票证券主表：5,538个代码、5,876个上市/退市事件，交易所与聚宽交叉核对，不提前暴露退市日。
- PIT财务：261,979行、5,217个资产，覆盖2004-03-31至2026-06-30。
- 现金分红事件：51,914行、5,380个资产，覆盖2000-08-23至2026-07-17。
- 事件化ETF主表：1,701个产品、1,824个上市/退市事件，其中123个退市事件。
- 中美利率：18,391行、3条序列，覆盖2000-01-03至2026-05-26。

仍然阻断正式验证的输入和流程包括：

- 没有历史行业成员、历史ST/停牌/涨跌停、可核验复权因子和历史估值。
- 没有完整ETF跟踪指数变更、全收益价格、规模流动性、费率跟踪、分红事件和指数估值历史。
- 财务当前是最终快照按最晚来源日期保守延迟，不是逐版本修订库；分红来源缺少可靠派息日。
- 没有正式目标权重状态机、purge/embargo walk-forward、时期留出、行业留出和参数稳定性报告。

现有 2020-2026 条件诊断使用 2026 年当前成分回看历史，且 QFQ 价格不是历史时点可得版本，只能检查规则行为，不能计算或引用策略收益。

**影响**：这是阻断实盘晋级的首要问题。继续调 65 分、20%估值分位、回撤或企稳阈值，只会提高过拟合风险。

### P0-2：当前 V4 代码和报告未进入 Git

Git 当前只跟踪 503 个文件；`strategy_lab/long_hold_v4/`、`tests/`、`configs/`、报告和 CI 工作流仍处于未跟踪状态。原始数据被忽略是正确的，但核心研究代码未版本化意味着：

- 无法从提交恢复本次审查状态。
- 远端 CI 不会运行这套代码。
- 当前本机哈希只能证明文件在本次运行前后没有变化，不能替代版本历史。

**影响**：工程可复现性为 D。应在确认提交边界后建立独立基线提交，不能把原始数据或凭证一起提交。

### P0-3：旧因子工厂绩效存在确定性失真

旧因子工厂同时存在三个问题：

1. `portfolio_weights_from_scores` 先裁剪单标的权重，再重新归一化（`strategy_lab/multi_factor_research_framework.py:511-546`）；当入选数量乘以上限不足 100%，或权重不均衡时，重新归一化会再次突破 `max_weight`。
2. 默认标签由 20 日未来价格生成（`strategy_lab/a_share_low_cost_factor_builder.py:29-40`），但 walk-forward 按每日行计算收益（`strategy_lab/factor_factory_walk_forward.py:151-170`），并以 252 期复利（同文件 `333-340`）。这会把高度重叠的 20 日收益当成独立日收益。
3. `split_boundaries` 训练期直接贴着测试期（同文件 `104-125`），没有按标签期限执行 purge/embargo，训练集末端标签可跨入测试期。

**影响**：旧因子工厂所有年化、Sharpe、回撤和实验晋级结论失效。完整 synthetic smoke test 通过只证明代码可运行，不能修复统计口径。

### P0-4：V2.5-V3.8 部分旧 HIRSSM 结果存在跨时点排名污染

`crowding_score` 对整个资产-日期面板直接 `rank(pct=True)`，没有按日期截面或仅用过去窗口分组（`strategy_lab/hirssm_v2_model.py:245-258`）。未来样本的分布会影响较早日期的拥挤度分数。

**影响**：所有依赖该字段的旧版本结果隔离为不可晋级历史产物；不能与 V4 当前结果或基准收益比较。

### P1-1：Tushare 日线库广但不满足长期价值回测

当前 raw daily-only 库覆盖 2000-01 至 2026-05：6,889 个工作日文件，6,395 个文件有数据，494 个为空，共 16,988,946 行。正式 V3.45 验收仍为 `acceptance_pass=false`：

- `920489.BJ` 在 2014-06-18 和 2014-07-28 各有一条 `close < low` 的 OHLC 异常。
- 11,353 条原始收益跳变被 guard 标记，涉及 4,326 个资产。
- 仅有未复权 OHLCV；`adj_factor`、`stock_basic`、`daily_basic`、`index_weight` 和 `index_dailybasic` 均缺失。

**允许用途**：受 guard 约束的价格状态、成交活跃度和市场宽度研究。  
**禁止用途**：长期总收益、分红回报、低 PE 选股、退市处理、指数成分回测和模型晋级。

### P1-2：当前资产覆盖不完整

当前快照共 190 个资产：178 只股票和 12 只 ETF。

| 分组 | 总数 | 数据通过 | 耐久性通过 | 阻断 |
|---|---:|---:|---:|---:|
| 银行 | 42 | 38 | 22 | 4 |
| 保险 | 5 | 5 | 2 | 0 |
| 公用事业 | 131 | 36 | 9 | 95 |
| 红利 ETF | 12 | 5 | 3 | 7 |

股票有 99/178 因必需字段缺失而阻断，主要集中在公用事业的自由现金流分红覆盖、分红增长和利润增长字段。ETF 侧发现 54 只关键词候选，只深度处理前 30 只，18 只因跟踪指数映射未审查而登记为数据缺口。交通、通信、消费必需品尚未形成股票数据构建路径。

**影响**：当前结果只代表受限观察池，不代表完整 A 股高股息资产集合。

### P1-3：股票缺少 ETF 同等级的逐字段新鲜度

股票构建器把财务、分红、PE、PB、价格和国债日期取最大值写入一个行级 `available_date`（`strategy_lab/long_hold_v4/stock_snapshot_builder.py:438-447`）。ETF 已分别记录价格、净值、估值、规模、分红、费率和指数日期（`strategy_lab/long_hold_v4/etf_snapshot_builder.py:561-568`）。

**影响**：股票当前研究能够阻断缺字段和未来日期，但无法清楚区分“某一字段长期未更新”和“整行最新”。正式 PIT 面板应为关键股票字段保存独立 `available_date`、来源版本和修订标识。

### P1-4：多 Agent 只有治理外壳，没有独立执行

旧框架共有 12 个角色定义、95 个运行 manifest；治理检查当前通过，但原始检查发现的 727 个错误全部由 35 个历史路径的 legacy allowlist 分类为旧债务。`strategy_lab/agents/` 实际包含 70 个 JSON 和 19 个 Markdown，没有 Agent 运行代码。

V4 的九角色日志由 `_agent_log` 一次性生成（`strategy_lab/long_hold_v4/pipeline.py:454-503`）。契约能防止角色名单或字段缺失，却不能证明：

- 每个角色有独立上下文和输入白名单。
- 上游产物由独立进程签名并交接。
- 否决权来自独立审查而不是同一函数的条件分支。
- Agent 失败、超时、重试和版本漂移被运行时处理。

**影响**：治理文档可复用，但不能把日志数量解释成多 Agent 增强了预测能力。

### P1-5：CI 的当前数据 CLI 步骤可能“假绿”

工作流最后执行 CLI（`.github/workflows/long-hold-v4.yml:27-29`），但 GitHub 干净环境没有被忽略的 `data_raw`。CLI 设计为数据不足时输出 `CASH_DATA_BLOCKED` 并以 0 退出，因此该步骤不能证明真实数据链可用。

**建议**：CI 只运行确定性的 fixture 测试和编译；当前数据端到端检查另设本机或带缓存数据的受控任务，并断言 `system_status`、manifest 和哈希结果。

### P1-6：参数只有经济解释，没有样本外校准

65 分、估值 20%分位、行业 35%分位、股债利差 1.5%、三年回撤 15%-20%、企稳和 T 的 z-score/持有期门槛均为可解释先验，但没有样本外稳定性证据。当前中国 10Y 已接入；美国利率、汇率、信用、盈利周期等没有进入 V4 默认模型。

**影响**：参数只能用于形成可审计研究假设，不能解释为预测最优阈值。

### P1-7：账户闭环仍是人工研究账本

账户支持核心/T 子账、费用、部分成交、幂等回执、现金分红、红利税追缴、送转拆股、NAV 和两级回撤制动。仍不支持：

- 券商持仓、资金和成交自动对账。
- 配股、合并换股、退市现金回收等复杂公司行为。
- 外部入金/出金后的份额化净值。
- 交易所日历驱动的完整停牌、涨跌停和废单状态机。

**影响**：可用于人工纸面跟踪，不能作为券商执行或会计权威账本。

### P1-8：凭证应轮换并迁出明文 JSON

`configs/data_credentials.json` 已被 `.gitignore` 忽略且当前不在 Git 跟踪列表中，这是正确的。但凭证仍以本机明文保存，且用户曾在聊天中暴露 token/密码。

**建议**：立即轮换已暴露凭证，迁移到环境变量或 Windows Credential Manager；报告不记录任何明文值。

### P2-1：仓库规模和版本命名降低可维护性

本轮编译范围`strategy_lab/scripts/tests`共有230个Python文件；`strategy_lab`顶层仅HIRSSM脚本就有94个，版本产物与当前代码混放。当前V4已有18个模块、约7,147行，仍需要在大量旧代码中辨认默认入口。

**建议**：物理拆为 `current/`、`historical/`、`legacy_invalid/`，并为每类结果提供机器可读 `evidence_status`。不要继续以 V3.94、V3.95 方式复制完整脚本。

### P2-2：V4 配置校验仍不完整

`validate_config` 已检查顶层区块、权重和、非负佣金、分批顺序、总预算及订单有效期（`strategy_lab/long_hold_v4/core.py:116-155`），但没有系统检查所有百分比范围、阈值单调性、路径类型和行业/资产枚举。

**影响**：默认配置安全，但人工改配置可能在运行深处才失败。后续应补一个小型 schema 或集中范围检查，不需要引入大型配置框架。

### P2-3：旧 one-command 系统依赖启动目录

`quant_model_system.py` 和旧 smoke runner 将根目录写死为 `Path("Introduction-to-Quantitative-Finance")`。从仓库根目录启动时路径会重复，必须从父目录运行。完整 demo 约耗时 162 秒，也不适合作为每次快速回归测试。

**建议**：默认根目录改为 `Path(__file__).resolve().parents[1]`；把快速契约测试和慢速 synthetic demo 分开。

### P2-4：聚宽SDK存在上游兼容性预警

证券主数据实时重建在Python 3.14.3、NumPy 2.4.4和`jqdatasdk 1.9.8`下成功，但SDK的pickle兼容层会触发NumPy `VisibleDeprecationWarning`。本轮已把实际使用的`jqdatasdk==1.9.8`补入依赖清单，干净环境不再因漏装直接失败；上游兼容性风险仍存在。

**建议**：数据采集任务使用独立锁定环境；升级NumPy或SDK前必须重跑证券生命周期哈希、关键代码迁移样本和14项PIT专项测试。

## 7. 建议执行顺序

### Gate E1：冻结可恢复基线

1. 审查并提交 V4 代码、配置、契约、测试、工作流和本报告。
2. 原始数据、outputs、账户实际成交和凭证继续排除在 Git 外。
3. 标记旧因子工厂和 V2.5-V3.8 结果为 `legacy_invalid_performance`。

### Gate E2：建设正式 PIT 历史数据

1. 保留已通过的股票主表、PIT财务、现金分红和利率数据，并持续验证血缘哈希。
2. 获取历史行业、ST/停牌/涨跌停、PIT复权因子、历史估值和可核验股票总收益。
3. 获取ETF全状态主表、全收益价格、规模流动性、费率跟踪、分红和指数估值历史。
4. 将财务最终快照升级为逐版本修订历史，并补足分红派息日；对Tushare两条OHLC异常保持显式quarantine。

### Gate E3：只验证固定假设，不调收益

1. 固定当前参数作为先验，不用全样本选阈值。
2. 运行 anchored 5 年训练 + 1 年测试、purge/embargo、5/10/20bps 和无 T 消融。
3. 分银行、保险、公用事业、ETF、牛熊/利率状态报告收益、回撤、换手和失败案例。
4. 只有样本外证据通过后，才允许报告预期区间或改默认阈值。

### Gate E4：纸面账户和 Agent 运行时

1. 先做券商对账和复杂公司行为，不直接自动下单。
2. 真正需要多 Agent 时，再将数据、基本面、估值、验证、风险和报告角色拆为独立任务；每个任务只读取白名单输入，并签名产物。
3. Agent 数量不是目标；独立否决、证据隔离和可复现交接才是验收标准。

## 8. 最终决策

当前合理动作仍是保持 500,000 CNY 现金，原因有两层：
