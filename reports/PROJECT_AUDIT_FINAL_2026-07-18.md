### P0-2：当前 V4 核心工作仍未进入 Git

仓库当前跟踪503个文件，但V4核心代码、测试、配置和工作流在Git查询中仍为0个已跟踪文件；`git status --porcelain -uall`有751项记录，其中749项未跟踪、2项修改。

**影响**：本机 manifest 能证明当前文件在一次运行中的完整性，却不能从提交历史恢复本次状态，也不能让远端 CI 真正执行当前系统。工程可复现性仍为 D。

### P0-3：旧因子工厂绩效存在确定性失真

旧因子工厂至少有三处会使绩效失真：

1. 单标的权重裁剪后再次归一化，可能重新突破上限（`strategy_lab/multi_factor_research_framework.py:511-546`）。
2. 默认标签是未来 20 日收益（`strategy_lab/a_share_low_cost_factor_builder.py:29-40`），walk-forward 却把每个日期的重叠标签当作逐期组合收益（`strategy_lab/factor_factory_walk_forward.py:159-169`）。
3. 训练期与测试期直接相邻，没有按标签期限执行 purge/embargo（`strategy_lab/factor_factory_walk_forward.py:104-125`）。

**影响**：旧因子工厂所有年化、Sharpe、回撤和晋级结论失效。8 项 smoke test 通过只证明代码能跑，不能恢复统计有效性。

### P0-4：V2.5-V3.8 的拥挤度因子存在跨时点排名污染

旧 HIRSSM 的 `crowding_score` 对整个资产-日期面板直接排名，没有按日期截面或过去滚动窗口分组（`strategy_lab/hirssm_v2_model.py:258`）。未来样本分布会影响历史日期的分数。

**影响**：依赖该字段的旧版本结果只能保留为历史研究记录，不得与 V4 或基准做有效绩效比较。

## 3. 重要风险与局限

### P1-1：Tushare 日线已形成可接受处理集，但仍不是总收益数据

当前原始日线目录共有6,924个文件，其中6,430个是2000-01-04至2026-07-17的受治理交易日，另有494个早期空占位。V3.45.1历史验收批次覆盖其中6,889个文件至2026-05-28：隔离2条北交所OHLC异常后，16,988,944行进入`accepted_processed_daily_only`；随后35个交易日由刷新清单逐文件锁定哈希。

这两种状态必须同时保留：原始批次并非全净，处理后的受控范围可以用于价格状态和流动性诊断。但它仍只有未复权 OHLCV，不能直接计算长期总收益、股息回报或历史估值。

### P1-2：复权因子通过数据门，但仍有长停牌复核债务

股票复权事件链覆盖 5,537/5,537 只生命周期资产。49,776 个合格现金分红除权日中，49,769 个与因子事件同日匹配，匹配率 99.9859%，高于固定 99.5%门槛；7 个未匹配事件被显式保留，没有静默丢弃。

独立验证器扫描全部 6,889 个 Tushare 源文件和 16,988,946 行数据，完成 53,051 次因子跳变检查。连续交易区间内绝对跳变超过 30%的异常为 0；169 个大跳变全部发生在超过 10 个自然日的交易间隔后，标记为 `LONG_SUSPENSION_REVIEW_REQUIRED`。

**边界**：因子可以进入历史数据层，但 169 个长停牌/重上市事件仍需结合公司行动复核；数据通过不等于模型可以晋级。

### P1-3：股票分红和证券生命周期已修正，但来源还不够完整

本轮把证券代码迁移登记扩展为两组：[`300114 -> 302132`](https://static.cninfo.com.cn/finalpage/2025-02-15/1222544408.PDF) 和 [`600849 -> 601607`](https://static.sse.com.cn/cs/zhs/scfw/gg/ssgs/2011-03-09/601607_2010_n.pdf)。迁移前后生命周期不重叠，基本面和行业记录按当时有效代码重分配，并在新代码启用日建立可用快照。分红链共重分配 13 条事件、删除 2,040 条生命周期外事件，同时排除 CDR `689009`；按统一市场截止日 2026-07-17 重建后的分红表为 49,874 行、5,083 个资产。

两次迁移均有交易所或法定披露平台证据，但当前迁移登记表不保证穷尽全部历史改码、吸收合并和重新上市事件。分红源仍缺少可靠派息日及完整公告修订版本，复权因子也主要依赖单一因子供应源。正式全收益链仍需要第二来源抽查和公司行动版本管理。

### P1-4：当前观察池不代表完整高股息 A 股

当前快照有178只股票和17只ETF。195只中85只通过数据门，110只阻断，36只通过长期耐久性门。股票主要覆盖银行、保险和公用事业；交通、通信、消费必需品等稳定分红行业尚未形成同等级构建链。

ETF关键词候选有54只，深度处理30只并形成17只完整当前指数映射快照；13只仍有来源或历史深度缺口。17只中只有6只通过模型数据门，其余11只主要因ETF自身历史不足三年而阻断。跟踪指数历史观察集仅461条，覆盖ETF生命周期主表的27.1%，且从2025年4月后才可见，因此不能回填历史。

### P1-5：股票缺少逐字段新鲜度

股票构建器仍将财务、分红、估值、价格和国债信息汇总为一个行级 `available_date`。ETF 已分别记录价格、净值、估值、规模、分红、费率、指数及全收益日期（`strategy_lab/long_hold_v4/etf_snapshot_builder.py:612-619`）。

**影响**：股票当前链能阻断未来日期和缺字段，却不能精确说明是哪一类信息陈旧。正式 PIT 截面应保存关键字段各自的可得日、来源版本和修订标识。

### P1-6：多 Agent 是治理结构，不是独立智能体运行时

旧框架有 12 个角色定义、95 个运行 manifest；治理自检通过。原始检查发现的 727 个错误全部被 35 个历史路径的 allowlist 归类为旧债务。V4 的九角色日志仍由同一流水线函数生成（`strategy_lab/long_hold_v4/pipeline.py:454-503`）。

它没有独立上下文、输入白名单运行时、独立签名交接、超时重试和真正独立的否决权。因此 Agent 数量不能解释为预测能力提高。

### P1-7：参数可解释，但没有样本外校准

耐久性 65 分、估值分位、股债利差、回撤、企稳、T 仓 z-score 和持有期都有经济解释，但尚无稳定性区间和样本外证据。中美 10 年利率已具备 PIT 数据资格，但尚未证明利率因子对本策略有增量价值。

### P1-8：账户是人工研究账本，不是券商级闭环

账户支持核心/T 子账、佣金税费、部分成交、幂等回执、现金分红、红利税、送转拆股、NAV 和回撤制动。仍不支持券商自动对账、配股/换股/退市回收、外部入金出金份额化、完整停牌/涨跌停/废单状态机。

### P1-9：本地明文凭证需要轮换

### P1-10：活跃股估值可交叉验证，退市股仍缺第二来源

### P2-1：仓库体量和版本复制降低维护效率

本轮审查范围共有258个Python文件。当前V4已扩展历史行业、基本面、交易状态、估值、ETF观察与指数注册表模块，但仍与大量V2/V3版本脚本和输出混放。版本号继续递增不能替代可恢复分支、变更记录和证据注册表。

### P2-2：上游 SDK 与长期运行环境仍需锁定

聚宽主数据采集在当前环境可运行，但 `jqdatasdk 1.9.8` 与新 NumPy 仍有兼容性预警。采集环境应单独锁定；升级 SDK、NumPy 或 Python 后必须重跑生命周期、代码迁移和 PIT 门禁测试。

### P2-3：旧策略脚本仍依赖特定启动方式

静态扫描发现 `strategy_lab` 顶层有 65 个文件包含对同目录模块的顶层绝对导入，共 73 处。它们通常可用 `python strategy_lab/xxx.py` 运行，却不保证支持 `python -m strategy_lab.xxx`。本轮已修复全局 Agent 门禁入口并增加回归测试；其余大多属于已隔离的旧 HIRSSM/V3 包装脚本，暂不做大范围机械改写。

**影响**：编译通过不能证明所有旧入口可按包模式启动。旧系统若需要复现，应先固定调用方式；正式 V4 新模块继续统一使用包导入。
