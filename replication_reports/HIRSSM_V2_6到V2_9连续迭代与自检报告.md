# HIRSSM V2.6 到 V2.9 连续迭代与自检报告

日期：2026-05-25

## 本轮目标

在 V2.5 已证明“组合风控可以显著降低回撤但会牺牲收益”的基础上，继续迭代到 V2.9：

- V2.6：把全组合风控改为局部 sleeve 风控，并增加状态/sleeve 归因。
- V2.7：加入回撤后的规则化再入场机制，减少过度现金拖累。
- V2.8：保留 style 核心暴露，对 industry 风险做更局部的降权。
- V2.9：固定混合 V2.4 收益基线与 V2.8 风险覆盖版，提升收益保留。

所有版本均必须通过：

- Python 编译。
- JSON 配置解析。
- smoke test。
- 5/10/20/30bps 输出。
- 与 V2.4 同口径对比。
- 自检报告。
- 归因报告。

## 小版本修复

V2.9 初次运行失败：

- 年化接近 0。
- 平均现金 93.34%。
- smoke test 显示 `no_missing_asset=False`。

定位：

- 固定混合权重时，非现金资产行丢失 `asset` 字段。
- 回测无法把这些权重映射到真实资产收益，导致组合实际接近全现金。

修复：

- 在 `blend_targets` 中显式写回 `template["asset"] = asset`。
- 该修复相当于 V2.9.1 小修复。
- 修复后重跑完整 V2.6-V2.9 链路，V2.9 通过自检。

## 效果评判

当前综合最优候选是 V2.7：

- 年化收益仅比 V2.4 低 0.32pct。
- Sharpe 提高到 0.422。
- 最大回撤改善约 10.95pct。
- 平均现金 21.51%，没有变成高现金策略。

V2.6 更偏防守：

- 最大回撤改善最大，达到约 15.70pct。
- 但收益牺牲比 V2.7 更大。

V2.8 没有优于 V2.7：

- style 核心保护和 industry 局部降权没有带来更好的收益/回撤交换。
- 保留为研究版本，不建议晋级。

V2.9 是收益保留折中版：

- 年化收益最高，接近 V2.4。
- 但回撤改善弱于 V2.7，Sharpe 也低于 V2.7。
- 适合作为“收益优先但仍要降回撤”的候选，而不是稳健默认候选。

## 自检结论

通过：

- `python -m py_compile strategy_lab/hirssm_v2_5_portfolio_risk_overlay.py strategy_lab/hirssm_v2_6_to_v2_9_risk_iteration.py`
- JSON 配置解析。
- V2.6/V2.7/V2.8/V2.9 smoke test。
- V2.6/V2.7/V2.8/V2.9 self-check。
- V2.9 小修复后 `no_missing_asset=True`。

仍需注意：

- V2.6-V2.9 只是改善组合风控层，不能消除 V2.4 底层 alpha 的 PBO 风险。
- 后续默认模型应暂定为“V2.7 风控候选”，但仍需做 PBO/DSR、年份失败案例和真实交易约束复核。

## 下一步

V3.0 或 V2.10 不应继续做阈值微调。

更有价值的方向：

- 对 V2.7 做年度失败案例复盘。
- 对 V2.7 做 5/10/20/30bps 下的换手和容量约束检查。
- 把 V2.7 的风控触发写成可解释的实盘监控信号。
- 对底层专家重新做 PBO/DSR，而不是只优化组合层。
