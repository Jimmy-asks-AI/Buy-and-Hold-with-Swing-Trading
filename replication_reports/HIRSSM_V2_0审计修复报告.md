# HIRSSM V2.0 审计修复报告

日期：2026-05-24

## 审计目标

检查 HIRSSM V2.0 是否存在会扭曲回测结论的重大实现错误，并区分两类问题：

- 必须直接修复的硬错误：回测口径、收益统计、组合治理。
- 导致表现不佳但不宜用全样本强行调参的问题：因子失效、专家拖累、仓位预算取舍。

## 已修复的重大错误

### 1. 回测起点错误

原问题：净值从 2000-01-05 开始，但模型首次实际目标权重为 2002-02-28，第一笔可执行交易日为 2002-03-01。策略早期空仓期被纳入年化统计，而基准也从不一致口径累计。

修复：`run_backtest` 现在从第一笔可执行交易日开始计算策略和基准净值。

影响：10bps 情景年化收益从旧口径约 5.74% 修正为 7.26%。

### 2. 总收益统计漏掉首日/首笔成本

原问题：`summarize_nav` 使用 `final_nav / first_nav - 1`，等于把第一天净值变化排除在总收益外；基准也同样存在这个问题。

修复：总收益改为 `final_nav - 1`，最大回撤路径显式补入初始净值 1.0。

### 3. 年度收益统计口径错误

原问题：年度收益使用 `year_end_nav / first_nav_of_year - 1`，会漏掉当年第一天收益。

修复：年度收益改为逐日收益复利：`prod(1 + daily_return) - 1`。

### 4. 失败专家仍默认启用

原问题：`range_reversal` 在专家 RankIC 和消融中均失败，但仍参与默认组合。

证据：

- style `range_reversal_score` RankIC 均值：-0.0476
- industry `range_reversal_score` RankIC 均值：-0.0213
- 旧模型中禁用该专家后年化和 Sharpe 均提高

修复：在 `configs/hirssm_v2_default.json` 中将 `range_reversal` 放入 `disabled_experts_by_default`，降级为观察专家。

## 修复后关键结果

回测起点：2002-03-01  
回测终点：2026-05-22  
默认禁用专家：`range_reversal`

10bps 成本情景：

- 总收益：446.75%
- 年化收益：7.26%
- 年化波动：19.72%
- Sharpe no RF：0.368
- 最大回撤：-54.81%
- 平均现金权重：23.08%
- 基准中证全指年化收益：8.02%
- 基准最大回撤：-71.48%

解释：修复后策略的绝对年化仍略低于基准，但波动和回撤明显低于基准，Sharpe 高于基准。

## 仍导致表现不佳的具体问题

### 1. 风格趋势和相对强弱专家方向偏弱

style 层 RankIC：

- `trend_expert_score`：-0.0295
- `relative_strength_score`：-0.0372
- `liquidity_score`：-0.0469

这说明在当前风格指数池中，月频趋势/相对强弱并没有稳定预测未来 21 日收益，甚至略偏反向。

### 2. 行业趋势信号接近无效

industry 层 RankIC：

- `trend_expert_score`：0.00027
- `relative_strength_score`：-0.0028

行业轮动收益主要没有来自趋势专家，而更多来自风险预算、行业流动性确认和配置结构。

### 3. 流动性 overlay 对总组合有拖累

消融显示：在禁用 `range_reversal` 的基础上，再禁用 `liquidity_overlay`，10bps 年化收益从 7.26% 提升到 7.97%，Sharpe 从 0.368 提升到 0.404。

原因判断：当前流动性确认在行业层略有效，但在 style 层明显负向；统一加到所有资产会污染风格选择。

### 4. 趋势专家对当前组合也有拖累

消融显示：再禁用 `trend_continuation` 后，年化收益提升到 7.89%，Sharpe 提升到 0.426。

原因判断：A 股指数层面的月频趋势信号容易在震荡市追涨杀跌，需要按状态和资产类型拆分，而不能统一使用。

### 5. 组合目标更偏控制回撤，不是收益最大化

策略平均现金权重为 23.08%；在 `risk_off_decline` 状态下平均现金权重为 42.84%。这降低了最大回撤，但也牺牲了牛市中的收益弹性。

### 6. 防御 sleeve 会持有负最终 alpha 的防御资产

最新一期 `000016` 上证50最终分数为负，但仍因防御 sleeve 获得约 6.77% 权重。这个设计符合防御预算逻辑，但会在防御资产自身弱势时拖累收益。后续应给防御 sleeve 增加现金替代门槛。

## 不建议立即做的事

- 不建议仅根据全样本消融把趋势和流动性全部禁用，这容易变成事后调参。
- 不建议直接提高权益仓位追赶基准收益，除非目标函数从“回撤控制”改为“收益最大化”。
- 不建议启用 ML ranker，当前样本数量和资产池规模仍不足。

## 下一步修复建议

1. 将 liquidity overlay 拆成 style/industry 两套，style 默认关闭，industry 保留观察。
2. 趋势专家改为状态条件化：只在 `risk_on_trend` 或行业宽度扩散时启用。
3. 防御 sleeve 增加现金替代门槛：防御资产最终分数为负且风险收缩不显著时转为现金。
4. 增加 walk-forward RankIC 门控，替代全样本人工禁用。
5. 增加样本外分段报告，避免用 2002-2026 全样本消融直接决定默认参数。
