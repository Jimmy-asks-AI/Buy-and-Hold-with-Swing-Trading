# Long Hold V4 订单与成交安全契约

## 适用范围

本契约只覆盖纸面订单、模拟成交和本地账户记账，不接券商，也不改变选股、入场、仓位或 PIT 门禁。订单 CSV 是不可变的授权信封；实时生命周期保存在本地 `order_state.json`，不能通过改写订单 CSV 改变。

## 威胁模型

执行端按失败关闭处理以下风险：

- 订单生成后，股数、价格、权重、有效期、账户版本或风险授权被修改；
- 旧运行的分阶段订单与新运行订单叠加成交；
- 已过期、已成交、已取消或已被取代的订单再次成交；
- 手工成交绕过订单授权，或风险覆盖权限被临时添加；
- 订单中的 T 仓持有天数被伪造；
- 成交价显著偏离研究时的参考价格；
- 单笔成交合法，但成交后的组合突破现金、资产、行业、核心仓、T 仓或持仓数量上限；
- 账户已写入而成交账本或订单状态尚未写入时进程中断。

本地 SHA-256 用于完整性与可追溯性，不是数字签名，不能抵御能够同时修改代码、订单、账户和全部哈希的主机管理员。真实券商认证、密钥托管和外部不可变审计日志仍不在本阶段范围内。

## OrderEnvelope v1

机器可读 schema：

- `data_catalog/schemas/long_hold_v4_order_envelope_v1.schema.json`
- `data_catalog/schemas/long_hold_v4_order_state_book_v1.schema.json`

规范化哈希覆盖除 `order_sha256` 自身外的全部字段，包括：

- `run_id`、运行清单哈希、配置哈希和交易日历哈希；
- 账户版本与账户状态哈希；
- 信号日、起始日、失效日、标的、sleeve、方向、股数和参考价格；
- 最大成交价偏离、目标权重、核心仓比例、T 仓持有期；
- 风险状态、风险覆盖授权、人工批准要求、意图状态和原因。

执行前会重新规范化并计算哈希。保留原 `order_id` 而修改任一字段会被拒绝；重新计算订单哈希也无法绕过 `order_state.json` 中已登记的原始哈希。

## 生命周期

允许状态为：`ACTIVE`、`PARTIALLY_FILLED`、`FILLED`、`CANCELLED`、`EXPIRED`、`SUPERSEDED`。

- 新研究运行会取代此前所有未完成订单；同资产、同 sleeve 的新订单会记录替代关系。
- 部分成交只能继续使用同一订单剩余股数，累计成交不能超过授权股数。
- 账户事件、NAV 更新或成交导致账户版本变化后，未开始成交的旧订单失效；已部分成交的同一订单可在剩余授权内继续，但仍要重新通过成交后组合约束。
- 手工批准不再是无订单通道。核心仓卖出和风险覆盖都必须先写入已哈希的授权信封。

## 成交检查

执行端使用本地交易日历重新计算 T 仓持有交易日，不读取订单 CSV 中的结果。每笔成交依次完成：

1. 订单、运行、配置、账户和日历绑定校验；
2. 生命周期、有效期、剩余股数、人工批准和风险覆盖校验；
3. 成交价相对 `indicative_price` 的偏离校验；
4. 结算规则与 T 仓实际持有交易日校验；
5. 在模拟成交后的账户上重新计算全部硬约束；
6. 账户、成交账本和订单状态作为一个可恢复写入集提交。

全持仓估值缺少任一价格时，成交阻断。执行估值快照至少包含 `asset`、`price`、`as_of_date`，日期不得晚于成交日，也不能超过配置的价格新鲜度上限。

## 初始化和迁移

新账户必须显式初始化：

```powershell
python -X utf8 -m strategy_lab.long_hold_v4.execution `
  --initialize-account `
  --initial-as-of 2026-07-22
```

已有 v1/v2 账户只迁移结构，不改现金与持仓：

```powershell
python -X utf8 -m strategy_lab.long_hold_v4.execution --migrate-account-state
```

迁移会生成账户 schema v3 的 `state_version`、`state_sha256`、空订单状态账本和成交账本。若 `account.json` 缺失，研究与执行都会硬错误，不再自动创建 50 万元账户。迁移前应在仓库外备份真实本地账户；账户、订单状态、账本、事务日志和成交回执均不得提交到 GitHub。

## 崩溃恢复

执行提交前会写入同目录事务日志和暂存文件。若三个目标文件只替换了一部分，下次执行会验证预期哈希并补齐剩余文件；暂存文件缺失或哈希不符时硬错误，不猜测账户状态。测试通过模拟第二个文件替换时崩溃来验证恢复链。
