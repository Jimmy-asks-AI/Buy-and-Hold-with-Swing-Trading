## 第一段：先确认服务入口

不要只问“能不能给历史行情”。先问：

1. 我的账户能否申请国泰海通开放金融云、君弘富易专业行情、道合·云数据 API 或其他量化数据服务？
2. 个人/高净值客户能否购买研究用途的数据授权？如果不能，请转接智能交易、机构数据或数据产品销售团队。
3. 是否支持 API、SFTP、对象存储或一次性批量文件交付？终端查看但不能导出，不满足本项目要求。
4. 可否先提供数据字典、样例文件、历史覆盖清单、价格表和试用范围？

## 第二段：所需数据

要求从 2005-01-01 起，覆盖存续和退市证券，不得只给当前股票池：

1. A 股证券主表和状态历史：上市、停牌、复牌、ST、退市、终止现金事件。
2. 历史行业分类：分类标准版本、行业代码、公告/发布日、生效起止日。
3. 逐版本财务报表：报告期、首次公告日、修订公告日、版本号、原始值和调整值。
4. 股票历史估值：PE、PB、PS、股息率、市值、计算口径、当日可得时间。
5. ETF 跟踪基准变更：公告日、生效日、旧/新指数、基金合同版本。
6. ETF 全收益链：未复权成交价、现金分红、份额折算、合并、退市和终止现金。
7. ETF 历史份额、规模、成交额、换手率、管理费、托管费和其他费率。
8. ETF 跟踪误差/跟踪偏离：窗口、算法、披露日和方法版本。
9. 指数全收益历史：至少中证全指 `000985.CSI`，并覆盖策略引用的主要指数。
10. 指数历史估值：PE、PB、股息率、成分权重和方法版本。
11. 资产级交易日历、涨跌停状态和可执行开收盘价。
12. 公司行为及复权因子历史版本，能够证明某交易日当时可得。

## 第三段：每类数据都要问的时间字段

每张表必须能回答：

- 该记录对应哪个业务日期或报告期？
- 首次对外可得日期/时间是什么？
- 后续何时修订，修订号是什么？
- 旧版本能否保留和回放？
- 当前最终值是否覆盖了历史版本？
- 退市证券和历史成分是否完整保留？

关键字段建议统一为：

```text
source_row_key
asset
effective_date / effective_start / effective_end
announcement_date
available_date
revision_id
observed_at
provider
methodology_version
```

如果供应商只能提供当前最终快照，或把当前行业/当前成分回填到历史，该数据只能用于当前研究，不能进入正式回测。

## 第四段：授权与技术问题

请对方书面确认：

1. 授权主体、账户类型和最低资产/服务费门槛。
2. 是否允许本地下载、长期存储、备份、哈希、重复回放和内部量化研究。
3. 是否允许保存历史版本，以及服务终止后是否必须删除。
4. 是否允许保存派生因子、目标权重、回测结果和审计清单。
5. 原始数据、派生数据和绩效结果分别能否对外展示或上传私有 Git 仓库。
6. API 的频率、并发、单次行数、每日流量和历史回溯限制。
7. 数据修订、缺失补发、接口变更和服务中断的 SLA。
8. 是否提供数据字典、字段单位、复权口径、时区、交易日历和错误码。
9. 是否可提供供应商交付批次号或文件校验值。
10. 费用、试用期、合同期限和增购模块价格。

## 收到回复后的验收

先索取少量样例，不急于购买。样例至少选择：

- 一只长期存续股票；
- 一只曾更名或变更行业的股票；
- 一只退市股票；
- 一只发生分红/份额调整的 ETF；
- 一个价格指数和对应全收益指数；
- 一份发生财务修订的年报。

用样例验证：

- 历史版本能否重建；
- `available_date` 是否真实；
- 退市和修订是否保留；
- 全收益恒等式是否成立；
- API/文件能否稳定导出；
- 授权条款是否允许本项目的存储和回放。

样例验收通过后再决定购买。仅有实时行情、当前财务快照、终端截图或不可导出的研究页面，不能关闭工作包 5 的数据阻断。
