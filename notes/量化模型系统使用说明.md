# 量化模型系统使用说明

更新日期：2026-05-23

## 系统入口

主入口：

```text
strategy_lab/quant_model_system.py
```

它把以下模块串成一条完整链路：

- 低成本因子计算器：`a_share_low_cost_factor_builder.py`
- 因子注册表审计：`factor_registry_audit.py`
- walk-forward 因子工厂：`factor_factory_walk_forward.py`
- 实验账本：`factor_factory_ledger.py`
- 底层因子研究函数：`multi_factor_research_framework.py`

## 一键 demo

运行：

```text
python -X utf8 Introduction-to-Quantitative-Finance/strategy_lab/quant_model_system.py demo --root Introduction-to-Quantitative-Finance --output-name quant_model_system_demo
```

输出：

```text
outputs/quant_model_system_demo/
```

关键文件：

- `SYSTEM_RUN_SUMMARY.md`
- `panel_validation.csv`
- `registry_summary.csv`
- `panel_with_factors.csv`
- `walk_forward/walk_forward_performance.csv`
- `walk_forward/selected.csv`
- `walk_forward/weights.csv`
- `ledger_row.csv`

Smoke test：
```text
python -X utf8 Introduction-to-Quantitative-Finance/strategy_lab/run_quant_system_smoke_tests.py
```

每次修改 walk-forward、真实数据 panel 构建、纸面跟踪或系统总入口后，都应先运行该命令。它会检查交易约束、point-in-time panel 构建和一键 demo 输出是否仍然可用。

注意：demo 使用合成数据，只证明工程链路可运行，不证明真实市场可投。

## 使用真实数据

真实数据必须先整理成 panel CSV。字段要求见：

```text
data_catalog/factor_factory_data_contract.md
```

最低字段：

- `date`
- `asset`
- `fwd_return`
- `industry`
- `log_mkt_cap`
- 注册表中指向的因子原始列

运行真实 walk-forward：

```text
python -X utf8 Introduction-to-Quantitative-Finance/strategy_lab/quant_model_system.py walk-forward \
  --panel-csv path/to/panel.csv \
  --registry-csv Introduction-to-Quantitative-Finance/data_catalog/a_share_factor_registry_v0.csv \
  --config-json Introduction-to-Quantitative-Finance/configs/factor_factory_walk_forward_demo.json \
  --output-dir Introduction-to-Quantitative-Finance/outputs/real_run_001 \
  --experiment-id real_run_001 \
  --hypothesis "first real A-share low-cost factor walk-forward model"
```

系统会先做 panel 校验。若有 `fail` gate，会停止运行，不允许带病回测。

## 输出如何读

### `panel_validation.csv`

检查输入数据是否符合最低契约：

- 必需字段是否存在。
- 是否有重复 `date-asset`。
- 日期数量是否足够 walk-forward。
- 股票数量是否足够截面检验。
- 注册表因子列是否存在。
- 标签覆盖率是否足够。

### `walk_forward/selected.csv`

每个训练窗口中通过筛选的因子。关注：

- 每个 split 入选因子数量。
- 家族是否过度集中。
- 是否长期只选中同一类因子。

### `walk_forward/walk_forward_returns.csv`

每个测试期组合收益：

- `gross_return`
- `turnover`
- `cost`
- `net_return`

### `walk_forward/walk_forward_performance.csv`

总绩效，分 gross 和 net 两列。真实模型以 `net` 为准。

### `factor_factory_experiment_ledger.csv`

所有实验的账本。核心字段：

- `decision`
- `failed_gates`
- `selected_factors`
- `families`
- `sharpe`
- `sortino`
- `max_drawdown`
- `win_rate`

## 当前系统边界

已经具备：

- 因子注册。
- 数据校验。
- 单因子诊断。
- 冗余控制。
- walk-forward 因子选择。
- 家族合成。
- 目标权重生成。
- 交易成本扣减。
- 实验账本。

仍需增强：

- 真实 A 股数据接入。
- 停牌、涨跌停、ST、新股过滤。
- 行业和风格暴露约束。
- 成交额容量约束。
- 指数增强基准权重。
- 更严格的交易执行模拟。
- 纸面交易和实盘漂移监控。
