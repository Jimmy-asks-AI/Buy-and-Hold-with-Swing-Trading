# 量化模型系统：数据、执行与纸面跟踪说明

日期：2026-05-23

## 真实数据入口

真实 A 股数据可以有两种输入方式：

1. 已经整理好的 panel：直接调用 `quant_model_system.py walk-forward`。
2. 分表数据：先用 `a_share_panel_builder.py` 合并行情、财务、行业表，再送入模型系统。

最低字段：

- 行情表：`date`、`asset`、`close`、`volume`、`amount`、`market_cap`。
- 财务表：`asset`、`available_date`，以及 ROE、毛利率、负债率、现金流等财务字段。
- 行业表：`asset`、`available_date`、`industry`。

命令示例：

```text
python -X utf8 Introduction-to-Quantitative-Finance/strategy_lab/a_share_panel_builder.py \
  --market-csv data/market.csv \
  --financial-csv data/financial.csv \
  --industry-csv data/industry.csv \
  --output-csv Introduction-to-Quantitative-Finance/outputs/real_panel/point_in_time_panel.csv
```

## 执行约束

`factor_factory_walk_forward.py` 支持以下执行约束：

- `tradeable_col`：不可交易证券目标权重置零。
- `amount_col`：用成交额估算容量。
- `fund_size`：组合资金规模。
- `max_participation_rate`：单期最大成交参与率。

约束逻辑是先过滤不可交易资产，再按 `amount * max_participation_rate / fund_size` 限制单资产权重上限，最后重新归一化。它不是完整订单簿模拟，但能阻止明显不可成交的目标组合进入纸面跟踪。

## 纸面跟踪

当实验账本给出 `promote_to_paper` 时，系统会生成：

- `paper_state.json`
- `target_weights_latest.csv`
- `paper_monitoring_checklist.csv`

默认状态：

- `status=paper_tracking`
- `live_trading_allowed=false`

纸面跟踪阶段只允许观察模型稳定性、换手、持仓漂移、容量和回撤，不允许直接解释为实盘交易授权。

## 当前边界

这个系统已经具备研究到纸面跟踪的工程闭环，但真实可投结论仍依赖真实 point-in-time 数据和至少 3-6 个月纸面跟踪。任何合成数据 demo 只能证明工程路径可运行，不能证明 alpha 存在。
