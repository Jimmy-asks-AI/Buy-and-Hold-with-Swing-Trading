# 学习复习纠错协议

Date: 2026-05-22
Domain: learning system
Status: active protocol

## Purpose

整个资料库学习必须按批次推进。每一批不允许只做摘要，必须完成：

```text
学习 -> 复习 -> 反证 -> 纠错 -> 沉淀 -> 更新队列
```

## Batch Definition

一个学习批次可以是：

- 1-5 篇课程 Markdown。
- 1 篇 PDF 研报或论文。
- 1 个代码示例。
- 1 个主题集合，例如“因子评价”“回测配置”“技术指标”。

批次不能太大。若材料很长，拆成多批。

## Required Outputs

每批至少产出：

- 一份学习笔记。
- 一份复习纠错记录。
- 一个可复用知识点或模板。
- 队列状态更新。

如果材料包含可复现方法，还要产出：

- 复现计划，或
- 代码模板，或
- 回测/实验脚本。

## Auto-Continue Rule

从 2026-05-22 起，每轮学习完成后，不再等待用户提示“继续下一轮”。只要当前任务没有被用户暂停，就自动进入下一轮。

每轮结束时必须：

1. 更新学习队列状态。
2. 写入复习纠错日志。
3. 验证新增代码或文件。
4. 选定下一轮材料。
5. 直接开始下一轮读取。

只有遇到以下情况才停止：

- 需要用户提供数据、权限或方向选择。
- 工具或文件损坏导致无法继续。
- 当前轮已触及上下文/执行时间边界，需要汇总交接。
- 用户明确要求暂停。

## Learning Step

学习阶段只做提取，不急着下结论。

必须提取：

- 研究问题。
- 核心概念。
- 数据需求。
- 方法和公式。
- 可交易假设。
- 可复现路径。
- 风险和失败条件。

## Review Step

复习阶段检查自己是否真的理解。

必须回答：

- 这批材料解决什么问题？
- 它和已有知识有什么关系？
- 哪些概念被我混淆过？
- 哪些结论只是作者声称，还没有被验证？
- 哪些内容可以转成因子、信号、策略、风控或代码？

## Falsification Step

反证阶段主动找问题。

至少检查：

- 是否可能有未来函数？
- 是否依赖特定样本？
- 是否忽略交易成本？
- 是否只在某种市场环境有效？
- 是否把 Beta、行业或风格暴露当成 Alpha？
- 是否存在数据口径或发布时间问题？
- 是否存在过拟合？

## Correction Step

纠错阶段要明确修改了什么。

类型：

- `concept_fix`：概念边界修正。
- `method_fix`：方法理解修正。
- `code_fix`：代码或模板修正。
- `queue_fix`：资料分类或优先级修正。
- `skill_fix`：需要同步进 skill 的通用规则。

每个纠错项必须写：

```text
原理解：
问题：
修正后：
影响：
```

## Promotion Step

只有满足下面条件，才能把知识升入通用 skill：

- 与单一策略无关，或已经明确标注适用边界。
- 有清晰概念边界。
- 有失败条件。
- 能复用于未来研究。
- 不依赖未验证的历史结论。

## Queue Status

学习队列状态统一使用：

- `new`：未读。
- `learning`：本批正在读。
- `learned`：已完成学习笔记。
- `reviewed`：已完成复习。
- `corrected`：已完成纠错。
- `promoted`：已同步到 skill 或通用模板。
- `deferred`：暂缓。
- `rejected`：不进入后续学习。
