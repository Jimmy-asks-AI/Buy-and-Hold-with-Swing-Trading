## 核心结论

这批材料展示了从技术指标到信号验证的最小闭环：

```text
日线行情 -> 技术指标 -> 买卖信号 -> 未来收益标签 -> 分组统计
```

它不是完整回测。完整回测还需要持仓、资金曲线、交易价格、手续费、滑点、停牌、涨跌停、仓位、基准和风险控制。

`technical.py` 中的 KDJ 示例使用过去窗口计算 `LOW_N`、`HIGH_N`、`RSV`、`K`、`D`，再用 K/D 金叉死叉产生 `signal=1/0`。这部分是可在交易时点计算的信号。

`main.py` 中的未来 1/5/10/20 日收益是评价标签，不是信号输入。只要未来收益不进入指标或信号公式，就不是未来函数。

## 公式算子

技术指标公式可转成 pandas 运算：

- `REF(X,N)` -> `shift(N)`
- `SUM(X,N)` -> `rolling(N).sum()`
- `MAX(X,N)` -> `rolling(N).max()`
- `MIN(X,N)` -> `rolling(N).min()`
- `MA(X,N)` -> `rolling(N).mean()`
- `EMA(X,N)` -> `ewm(span=N).mean()`
- `WMA(X,N)` -> rolling weighted average
- `IF(COND,A,B)` -> `loc` 或 `where`

这些算子本身不构成策略，只有放入可验证假设、信号、仓位和回测规则后才形成研究对象。

## 125指标目录

Excel 中收录了 125 个技术指标，字段包括指标名称、计算公式、指标描述、买卖信号。它适合作为技术指标翻译练习和候选假设库，但不适合机械网格搜索。

风险：

- 125 个指标乘以多个参数，很容易产生严重数据挖掘。
- 交叉、阈值、窗口参数若反复优化，必须做样本外和 PBO。
- 不同指标高度相关，不能把数量当作独立证据。
- 部分指标信号描述是经验规则，不等于有统计预测能力。

## 因子卡模板

原模板包含：

```text
Domain, Universe, Frequency, Data Fields,
Rationale, Formula, Preprocessing,
Evaluation, Robustness, Failure Modes, Implementation Notes
```

本轮修正：后续因子卡应额外明确：

- 信息可得时点。
- 因子方向。
- 标签窗口。
- 交易成本和容量假设。
- 样本外验证。
- 与已有因子的相关性。
- 是否做过 PBO 或参数敏感性。

## 复习与纠错

- 纠错1：`main.py` 是信号验证，不是完整回测；不能用它的输出直接判断策略可交易。
- 纠错2：未来 N 日收益用于评价是合理的，但最后 N 天缺失收益不应强行填成 False，应在统计时 drop missing。
- 纠错3：技术指标库是候选假设库，不是自动 alpha 库；多指标多参数必须防止过拟合。
- 纠错4：KDJ 平滑口径需要明确，不同软件对 SMA/EMA 的实现可能不同。
- 纠错5：因子卡必须补充数据时点、方向、样本外、成本和失败模式，否则不能进入策略库。
