## One-Line Takeaway

KDJ 示例教会“数据如何变成事件信号”，低 PB 示例教会“资产如何按因子横截面排序”；两者都只是研究闭环的前半段，必须再进入组合、成本、基准和风控。

## 1. KDJ 信号验证

KDJ 示例流程：

```text
读取股票数据
-> 计算 LOW_N/HIGH_N/RSV/K/D
-> 生成金叉/死叉 signal
-> 计算未来 1/5/10/20 日收益
-> 按 signal 分组统计未来收益
```

它能回答：

```text
金叉或死叉出现后，未来 N 日收益分布是否不同？
```

它不能回答：

```text
按这个信号真实交易，资金曲线如何？
最大回撤多少？
扣交易成本后是否有效？
多只股票同时发信号如何处理？
和哪个基准比？
```

所以它是 `signal validation`，不是 `full backtest`。

## 2. 从信号到因子

KDJ 思维：

```text
某个事件是否发生？
事件后收益是否不同？
```

因子思维：

```text
同一时点，给一批资产打分。
高分资产未来是否更好？
排序关系是否稳定？
```

两种研究的差别：

| 维度 | KDJ 信号验证 | 因子研究 |
| --- | --- | --- |
| 输出 | 事件信号 | 横截面分数 |
| 典型问题 | 信号后未来收益如何 | 高分组是否跑赢低分组 |
| 检验方法 | 事件后收益统计 | IC、Rank IC、分组收益 |
| 下一步 | 策略回测 | 组合回测 |

## 3. 低 PB 因子

研究问题：

```text
在 A 股市场中，低 PB 股票未来是否比高 PB 股票表现更好？
```

规范表达：

```text
每月底在可交易 A 股股票池中计算 PB，用 -PB 作为价值因子分数。
将股票按因子分成 5 组，观察未来 1 个月收益、IC、Rank IC 和多空收益。
```

因子卡：

```text
因子名称：低 PB 因子
因子类型：价值
公式：value_score = -PB
方向：value_score 越高，代表 PB 越低
频率：月度
未来收益窗口：未来 20 个交易日
逻辑：低估值可能存在均值回归或估值修复
风险：价值陷阱、行业集中、盈利恶化、金融地产周期暴露
```

## 4. 因子研究标准流程

```text
1. 定义股票池或资产池。
2. 明确数据字段和时间口径。
3. 写因子公式和方向。
4. 计算调仓日因子值。
5. 计算未来收益标签。
6. 计算 IC 和 Rank IC。
7. 做分组收益。
8. 检查多空收益和多头组相对基准。
9. 检查换手、成本、行业、市值和流动性暴露。
10. 决定是否进入完整组合回测。
```

## 5. 回测配置标准

没有配置的回测不能信。

最小配置必须包含：

- 策略名称。
- 研究问题。
- 资产范围。
- 样本区间。
- 样本内/样本外。
- 数据频率。
- 调仓频率。
- 调仓日。
- 买入规则。
- 卖出规则。
- 持仓数量。
- 权重规则。
- 成交价假设。
- 交易成本。
- 滑点。
- 涨跌停处理。
- 停牌处理。
- 新上市和退市处理。
- 基准。
- 输出指标。

## 6. 技术指标公式转代码

常见映射：

```text
REF(X,N) -> shift(N)
SUM(X,N) -> rolling(N).sum()
MAX(X,N) -> rolling(N).max()
MIN(X,N) -> rolling(N).min()
MA(X,N) -> rolling(N).mean()
EMA(X,N) -> ewm(span=N, adjust=False).mean()
CUMSUM(X) -> cumsum()
ABS(X) -> abs()
```

注意：技术指标公式语言和 pandas 实现并不总是一一等价，尤其是 `SMA`、`DMA`、初始化值、缺失值、除零和复权口径。

## 8. Falsification

主动反证：

- KDJ 代码若直接用于交易，会缺少仓位、成本和基准。
- 当前资料目录下未发现 `股票数据/` 子目录，因此无法立即复现原始 KDJ 输出。
- 低 PB 因子可能只是行业暴露、周期暴露或价值陷阱，不应直接进入策略。
- 因子验证若用财报报告期数据而非公告日，会产生未来函数。
- 月末因子若用月末收盘后才知道的数据，则成交应至少滞后到下一交易日。

## 9. Corrections

### Correction 1: KDJ 示例不是策略

原理解：

```text
KDJ 金叉死叉可以作为策略。
```

问题：

```text
原代码只统计未来收益，没有完整交易规则。
```

修正后：

```text
KDJ 示例是信号验证。只有补齐仓位、成本、成交、持仓、基准和净值曲线，才接近完整策略回测。
```

影响：

```text
后续所有技术指标材料先进入 signal validation，再决定是否进入 backtest。
```

### Correction 2: 技术指标说明不能逐字视为严谨实现

原理解：

```text
说明里的公式到 pandas 代码都可直接照搬。
```

问题：

```text
`SUM(成交量,10)` 示例列名写成“10日收盘价总和”，列名与内容不一致。
`DMA(X,a)` 示例中 a=10，不符合常见 DMA 权重系数 0<a<1 的含义。
`SMA(X,N,M)` 用 `ewm(span=(N-M))` 近似表达，未严格证明等价于公式 alpha=M/N。
```

修正后：

```text
技术指标公式材料只能作为转换参考。每个公式进入研究前必须单独校验定义、边界条件、初始化和缺失值。
```

影响：

```text
后续技术指标库必须增加 formula_audit 字段。
```

### Correction 3: 低 PB 的“便宜”需要风险拆解

原理解：

```text
低 PB 表示便宜。
```

问题：

```text
低 PB 也可能来自盈利恶化、资产质量差、行业衰退或市场长期折价。
```

修正后：

```text
低 PB 只是价值候选因子，必须检查行业、市值、盈利质量、流动性和价值陷阱。
```

影响：

```text
低 PB 不能单独作为策略，需要与质量、风险和组合约束联合验证。
```
