## One-Line Takeaway

量化研究不是用代码寻找稳赚公式，而是把投资假设转成可检验、可复现、可比较、可复盘的证据链。

## Core Mental Model

```text
投资想法
-> 可验证问题
-> 数据定义
-> 指标/因子构造
-> 信号生成
-> 组合和仓位
-> 交易成本和约束
-> 回测检验
-> 风险归因
-> 样本外跟踪
-> 结论和迭代
```

研究的第一目标不是找到最高收益，而是判断一个结论是否可靠。收益只是一部分证据，必须和风险、成本、换手、容量、稳定性、可解释性一起看。

## Vocabulary

### 数据

数据是研究原材料，必须同时回答：

- 这个数字是什么意思？
- 这个数字什么时候产生？
- 当时的投资者能不能看到？
- 这个数字能不能用于交易决策？

常见数据：

- 行情：开盘、最高、最低、收盘、成交量、成交额、换手率。
- 基本面：收入、利润、现金流、ROE、负债率。
- 估值：PE、PB、PS、股息率、EV/EBITDA。
- 预期：一致预期、评级、目标价、覆盖数量。
- 另类：新闻、公告、社交媒体、搜索、招聘、卫星、电商。

### 指标

指标是从原始数据加工出的数值，例如 KDJ、MACD、均线、PE、PB、ROE、波动率。

指标本身不等于因子，也不等于交易策略。

### 因子

因子是可能解释或预测未来收益的变量。它可以来自估值、质量、成长、动量、反转、波动、流动性、情绪、宏观、事件等维度。

一个因子必须回答：

- 为什么可能有效？
- 怎么计算？
- 数据在当时是否可得？
- 历史关系是否稳定？
- 是否能扣除成本后交易？

### 信号

信号是把指标或因子转成买入、卖出、持有、回避、加仓、减仓等动作提示。

信号验证常看“信号出现后未来 N 日收益”，但这不等于完整回测。

### 策略

策略是完整交易规则，至少包括：

- 买什么？
- 什么时候买？
- 买多少？
- 什么时候卖？
- 多久调仓？
- 成本怎么算？
- 哪些情况不能交易？
- 如何控制风险？

### 组合

组合是策略真正持有的资产及权重。选什么和买多少是两个问题。

组合构建需要处理：

- 等权、市值加权、风险加权。
- 单票上限。
- 行业暴露。
- 风格暴露。
- 换手控制。
- 流动性约束。

### 风控

风控不是消灭风险，而是让风险可理解、可承受、可复盘。

风险来源：

- 市场风险。
- 行业风险。
- 风格风险。
- 流动性风险。
- 模型风险。
- 数据风险。
- 执行风险。
- AI/Agent 风险。

## Research Standards

一个合格研究必须写清楚：

- 研究问题。
- 投资逻辑。
- 数据范围和时间口径。
- 因子/信号定义。
- 回测规则。
- 成本、滑点、交易限制。
- 绩效指标。
- 风险归因。
- 稳健性检验。
- 失败条件。
- 结论和下一步。

## Critical Failure Modes

### 未来函数

在历史某个时点使用了当时不可能知道的信息。常见形式：

- 用未来收益生成今天信号。
- 用财报报告期而不是公告日。
- 用现在的指数成分股回测过去。
- 用全样本均值/分位数做历史标准化。
- 看完整历史后挑最优参数。

### 数据口径错误

同名字段可能口径不同：

- 前复权、后复权、不复权。
- PE 静态、TTM、动态。
- 总市值、流通市值。
- 财报报告期、公告日。
- 是否包含 ST、停牌、退市。

### 回测假设不现实

常见问题：

- 收盘后才知道信号，却假设当天收盘价成交。
- 不考虑手续费、印花税、滑点、冲击成本。
- 涨停买入、跌停卖出没有限制。
- 停牌股票假设可交易。
- 忽略容量和成交额限制。

### 因子有效但策略无效

可能原因：

- 因子 IC 有效但太弱。
- 换手太高，成本吃掉收益。
- 因子只是行业、市值或 Beta 暴露。
- 有效性只在样本内。
- 极端分组有效，中间组不单调。
- 因子拥挤或环境变化。

## Minimum Research Loop

```text
1. Define:
   universe, time range, frequency, benchmark.

2. Build data:
   fields, timestamp, adjustment, tradability, missing values.

3. Build factor/signal:
   formula, direction, lag, preprocessing.

4. Build portfolio:
   selection rule, weighting rule, rebalance rule, constraints.

5. Backtest:
   execution price, cost, slippage, limit-up/down, suspension, delisting.

6. Evaluate:
   annual return, drawdown, volatility, Sharpe, IR, turnover, win rate.

7. Diagnose:
   year split, regime split, industry/style exposure, parameter sensitivity.

8. Decide:
   accept, modify, queue, or reject.
```

## Reusable Knowledge

- 先问数据时间口径，再谈模型。
- 信号验证不是完整回测。
- 因子研究不是只看收益曲线，要看 IC、Rank IC、ICIR、分组单调性和稳健性。
- 组合管理是独立问题，因子只负责排序，组合负责可持有。
- 风控要从研究阶段开始，不是实盘亏损后补救。
- AI 可以辅助写代码和抽取想法，但不能替代交易假设、数据口径和回测逻辑审查。
